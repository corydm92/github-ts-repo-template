# App Onboarding Guide

Use this when adding a new app under `/apps`.

## 1) Create the folder

```
/apps/<app-name>/
```

## 2) Add `package.json`

Required scripts (app-level gate):

- `ci` (must exist)
- `lint` (if applicable)
- `type-check` (if applicable)
- `test` (if applicable)
- `format` or `format:check` (if applicable)

Example:

```json
{
  "name": "@app/<app-name>",
  "private": true,
  "scripts": {
    "format": "prettier . --write",
    "lint": "eslint .",
    "type-check": "tsc -p tsconfig.json --noEmit",
    "test": "echo \"No test specified\" && exit 0",
    "ci": "pnpm run format && pnpm run lint && pnpm run type-check && pnpm run test"
  }
}
```

## 3) Add a README

Document:
- Purpose
- Entry points
- Required env vars
- Deployment target

## 4) Add app config as needed

Examples:
- `tsconfig.json`
- `eslint.config.js`
- `.env.example`

## 5) Verify CI wiring

From repo root:

```
pnpm run ci
```

If the app changed and its `ci` script exists, it will run.
